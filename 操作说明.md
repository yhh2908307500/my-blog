# Hugo 博客系统操作说明

## 项目概述

本项目是使用 Hugo 框架创建的静态博客系统，通过 GitHub Actions 自动部署到 GitHub Pages。

## 目录结构

```
├── .github/           # GitHub Actions 配置
├── archetypes/        # 内容模板
├── content/           # 博客内容
│   └── posts/         # 博客文章
├── public/            # 生成的静态文件
├── resources/         # 资源文件
├── themes/            # 主题
│   └── ananke/        # Ananke 主题
├── hugo.toml          # 配置文件
└── 操作说明.md        # 本操作说明
```

## 基本操作

### 1. 安装 Hugo

项目中已包含 Hugo 二进制文件，位于项目根目录。

### 2. 本地预览

```bash
# 启动本地服务器
./hugo server -D

# 访问地址
http://localhost:1313/my-blog/
```

### 3. 创建新文章

```bash
# 创建新文章
./hugo new content posts/文章标题.md

# 编辑文章
编辑 content/posts/文章标题.md 文件
```

### 4. 构建网站

```bash
# 构建网站
./hugo

# 构建结果在 public/ 目录
```

### 5. 部署到 GitHub

```bash
# 添加更改
git add .

# 提交更改
git commit -m "添加新文章"

# 推送到 GitHub
git push

# GitHub Actions 会自动部署
```

## 配置说明

### 1. 网站配置

编辑 `hugo.toml` 文件：

```toml
baseURL = 'https://yhh2908307500.github.io/my-blog/'
languageCode = 'zh-cn'
title = '我的博客'
theme = 'ananke'
```

### 2. 主题配置

Ananke 主题配置位于 `themes/ananke/` 目录。

## 部署流程

1. **代码推送**：推送到 `master` 或 `main` 分支
2. **自动构建**：GitHub Actions 自动构建网站
3. **部署**：部署到 GitHub Pages
4. **访问**：通过 https://yhh2908307500.github.io/my-blog/ 访问

## 常见问题

### 1. 部署失败

- 检查 GitHub Pages 是否启用
- 检查 GitHub Actions 工作流配置
- 查看 Actions 日志获取详细错误信息

### 2. 页面不显示

- 检查文章的 `draft` 设置是否为 `false`
- 检查 Hugo 版本兼容性
- 检查主题配置

### 3. 本地预览正常，部署后异常

- 检查 `baseURL` 配置是否正确
- 检查 GitHub Pages 部署状态

## 高级功能

### 1. 添加主题

```bash
# 添加新主题
git submodule add https://github.com/主题仓库.git themes/主题名

# 修改配置文件
theme = '主题名'
```

### 2. 自定义域名

1. 在 GitHub Pages 设置中添加自定义域名
2. 配置 DNS 记录
3. 更新 `baseURL` 配置

### 3. 添加评论系统

根据主题文档添加 Disqus、Utterances 等评论系统。
